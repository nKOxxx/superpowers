## Cron Job: Workflow Monitor

### Automatic Stuck Workflow Detection
- **Job ID:** `3b5a80d7-295e-4d6d-b16a-2b625b891a17`
- **Schedule:** Every 2 hours (`0 */2 * * *`)
- **Timezone:** Asia/Dubai (GMT+4)
- **Action:** Checks GitHub Actions for stuck workflows
- **Alert:** Notifies Nikola if any workflow runs >30 minutes

### How It Works
1. Queries GitHub API for recent workflow runs
2. Looks for `in_progress` status with runtime >30 min
3. Sends alert with link to cancel: https://github.com/nKOxxx/gulf-watch-v2/actions

### Why 2 Hours?
- RSS workflow runs hourly (takes ~2-3 min normally)
- If stuck, catches it within 2 hours max
- Balances monitoring vs API calls

## Severity Filter Feature

### Problem
298 incidents overwhelming users - too much noise, hard to find critical information.

### Solution: Smart Severity Scoring
**Calculate severity score (0-100) based on:**
1. **Government source** = 50 points (highest credibility)
2. **Credibility** = up to 30 points for non-government sources
3. **High-impact keywords** = up to 40 points:
   - missile, rocket, drone, air defense, interceptor, attack, strike, explosion, casualties, killed, wounded
   - 10 points per keyword match
4. **Recency** = up to 10 points:
   - <1h = 10pts, <6h = 7pts, <12h = 5pts, <24h = 3pts

**Critical threshold:** Score ≥ 60

### UI Changes
- **"Severity Filter"** section in sidebar
- Two buttons: "Show All" (default) and "🔴 Critical Only"
- Help text: "Critical = Government sources OR high-impact incidents"
- Updates incident count when filtering

### How It Works
- Click "Critical Only" → shows only incidents with score ≥ 60
- Click "Show All" → shows all incidents
- Works together with country filter (can filter by both)
- Real-time count updates
