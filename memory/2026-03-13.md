## Late Night Session (After Midnight GMT+4)

### The Crisis
- **Gulf Watch V2** (production) broke - showing "Failed to load regional data" error
- Mobile app using `gulfwatch.live` domain was broken
- Root cause: Regional panel code was accidentally deployed to production earlier today

### What Happened
1. Deployed regional panel UI to gulf-watch-v2 (production) instead of keeping it in testing
2. Mobile app (using gulfwatch.live) showed error overlay on regional map
3. User discovered the issue and was (rightfully) frustrated

### Actions Taken

#### 1. Reverted gulf-watch-v2
- Reset git to March 11 commit (`9461604`)
- Force pushed to main
- Removed all regional panel code from production
- Added FORCE_REDEPLOY.txt to trigger fresh Vercel build

#### 2. Fixed Vercel Deployment Limit
- Hit 100 deployments/day limit
- GitHub Actions running every hour = 24 deployments/day per repo
- Two repos = 48/day from cron alone
- Fixed: Reduced gulfwatch-testing to every 3 hours (8/day)
- Kept gulf-watch-v2 at hourly as user requested

#### 3. Fixed GulfWatch-Testing Issues
**Regional Panel Error:**
- Cause: Relative URL path `'data/regional_stats.json'` 
- Fix: Changed to absolute `/data/regional_stats.json`
- Added retry mechanism (retries once after 2 seconds)
- Added HTTP status checking
- Commit: `6bfe25a`

**Casualties Field Empty:**
- Cause: Current incidents.json generated BEFORE casualty extraction code
- Fix: Will populate on next GitHub Actions run (within 3 hours)
