# 2026-03-03 — Gulf Watch v2 Mobile Redesign

## Problem
Mobile version was completely unusable - sidebar took 50% of screen, buttons too small, no touch optimization.

## Solution: Complete Mobile Redesign

### New Mobile UI

#### Bottom Navigation Bar
- Fixed at bottom of screen (60px height)
- Two buttons: **Map** and **Incidents**
- Active state highlighted in red
- Touch-friendly (22px icons, large tap targets)
- Backdrop blur effect

#### Slide-Out Sidebar
- Hidden by default (off-screen to right)
- Slides in when tapping "Incidents" or pull tab
- Full-screen overlay with dark backdrop
- Swipe/tap overlay to close
- Smooth 0.3s transition animation

#### Pull Tab
- Visible handle at bottom of map
- Shows users they can pull up incident list
- Visual indicator (white bar)
- Tappable to open sidebar

#### Responsive Typography
- Header: 16px (was too big)
- Incident titles: 13px with 1.4 line height
- Badges: 8-9px (readable but compact)
- Meta text: 10px
- Touch-optimized padding

#### Touch Optimizations
- Buttons: Larger tap targets (44px min)
- Cards: Full-width, adequate spacing
- Filter buttons: Compact but tappable
- Scrollable incident list

### CSS Changes

```css
/* Mobile breakpoint: 768px */
@media (max-width: 768px) {
  /* Header: 50px compact */
  /* Map: Full screen minus header and nav */
  /* Sidebar: Slide-out from right */
  /* Finance panel: Hidden (map focus) */
}
```

### JavaScript Features

```javascript
// Mobile navigation
initMobileNav() {
  // Detect mobile viewport
  // Handle Map/Incidents toggle
  // Overlay click to close
  // Pull tab interaction
  // Window resize handling
}
```

### User Flow

1. **Open on mobile** → Shows full map
2. **Tap "Incidents"** → Sidebar slides in with incident list
3. **Tap incident** → Shows details, can click through to source
4. **Tap overlay or "Map"** → Returns to map view
5. **Pull tab** → Alternative way to open incident list
